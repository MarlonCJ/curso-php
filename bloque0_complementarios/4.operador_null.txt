
OPERADOR NULL ??

el operador ?? se conoce formalmente en programación como el Operador de Fusión Null (o Null Coalescing Operator en inglés). Fue introducido en PHP 7.0 y cambió por completo la forma en que escribimos código.

Aquí tienes una explicación detallada de cómo funciona y por qué es tan útil:

¿Qué hace exactamente?
El operador ?? evalúa el valor que tiene a su izquierda y toma una decisión simple:

Si el valor de la izquierda existe y NO es null, usa ese valor.

Si el valor de la izquierda no existe (está indefinido) o es null, entonces usa el valor que está a su derecha (como un "plan B" o valor por defecto).

La analogía del restaurante:
Imagina que vas a comer y le dices al mesero: "Tráeme la sopa del día ($sopa). Si no hay (??), tráeme una ensalada ('ensalada')".

El "Antes" y el "Después"
Para entender su magia, es útil ver cómo teníamos que escribir las cosas antes de que existiera este operador. Imagina que queremos asignar un nombre de usuario, y si no hay uno, llamarlo "Invitado".

Antes (La forma larga con isset):
Teníamos que usar el operador ternario (?:) combinado con la función isset() para evitar errores si la variable no existía.

PHP
$usuario = isset($_GET['usuario']) ? $_GET['usuario'] : 'Invitado';
Ahora (La forma elegante con ??):
El operador fusiona esa comprobación en dos simples caracteres.

PHP
$usuario = $_GET['usuario'] ?? 'Invitado';
Un detalle crucial: ?? vs ?:
Es muy común confundir el operador de fusión null (??) con el operador ternario corto (también conocido como operador Elvis ?:). Tienen una diferencia fundamental:

?? (Fusión Null): Solo se activa si la variable es null o no existe. Si la variable está vacía (como ""), es un cero (0) o es falsa (false), el operador dice: "¡Ah! Sí hay algo aquí, lo usaré".

?: (Operador Elvis): Se activa si la variable es cualquier valor "falso" (falsy). Esto incluye null, pero también "", 0, false y arreglos vacíos [].

Ejemplo práctico de la diferencia:

PHP
$cantidad = 0;

// Usando ?? (Fusión Null)
echo $cantidad ?? 10; 
// Resultado: 0 (Porque 0 no es null, es un número válido).

// Usando ?: (Elvis)
echo $cantidad ?: 10; 
// Resultado: 10 (Porque 0 se considera un valor "falso" o vacío).
¿Se pueden encadenar?
¡Sí! Esta es otra de sus grandes ventajas. Puedes poner varios en fila para comprobar múltiples variables hasta encontrar la primera que tenga un dato real:

PHP
// Usa el nombre que envió por formulario. Si no hay, usa el de la base de datos. Si tampoco hay, usa "Desconocido".
$nombreFinal = $_POST['nombre'] ?? $datosUsuario['nombre'] ?? 'Desconocido';
En resumen, el operador ?? te ayuda a escribir código más limpio, más fácil de leer y libre de los errores de "variable no definida".
